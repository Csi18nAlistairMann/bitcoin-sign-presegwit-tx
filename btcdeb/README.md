Kallewoof's debugger, found at https://github.com/kallewoof/btcdeb, allows the
user to single-step through a bitcoin transaction.

Simple usage: `btcdeb --txin <input-tx> --tx <new-tx>`

The entire command is generated using `python2 bsec-3374.py` - also in this
repository - which uses `input-tx` as generated by `generate-transaction.sh` to
create `new-tx`. The principle is that `new-tx` is signed by the private key
used to create `input-tx`.

An example run is shown below.

On the left side is the script described by the two transactions, on the right
side is the stack state. Type 'step' and press return, and the top item from
the left is removed and applied to the stack, and the new state of both
displayed. Press return - which repeats the last written command, here 'step',
and the new top of the script is removed, applied to the stack, and the new
state of both is applied. Keep pressing return and btcdeb steps through the
remainder of the script. Assuming the script passes, the left side ends up
empty and the right side ends up with "01". If the script does not pass,
btcdeb will display an error appropriate to it. Use ctrl-c to exit btcdeb.
